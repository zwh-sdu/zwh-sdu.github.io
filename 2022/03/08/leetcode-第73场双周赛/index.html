第一题-easy

2190. 数组中紧跟 key 之后出现最频繁的数字

题目描述

给你一个下标从 0 开始的整数数组 nums ，同时给你一个整数 key ，它在 nums 出现过。

统计 在 nums 数组中紧跟着 key 后面出现的不同整数 target 的出现次数。换言之，target 的出现次数为满足以下条件的 i 的数目：

  • 0 <= i <= n - 2

  • nums[i] == key 且

  • nums[i + 1] == target 。

请你返回出现 最多 次数的 target 。测试数据保证出现次数最多的 target 是唯一的。

样例

  • 示例1

  • 示例2

思路

​ 用一个map维护target出现的次数，最后选出value最大的数就是target。

第二题-mid

2191. 将杂乱无章的数字排序

题目描述

给你一个下标从 0 开始的整数数组 mapping ，它表示一个十进制数的映射规则，mapping[i] = j 表示这个规则下将数位 i 映射为数位 j 。

一个整数映射后的值为将原数字每一个数位 i （0 <= i <= 9）映射为 mapping[i] 。

另外给你一个整数数组 nums ，请你将数组 nums 中每个数按照它们映射后对应数字非递减顺序排序后返回。

注意：

  • 如果两个数字映射后对应的数字大小相同，则将它们按照输入中的 相对顺序 排序。

  • nums 中的元素只有在排序的时候需要按照映射后的值进行比较，返回的值应该是输入的元素本身。

样例

  • 示例1
  • 示例2

思路

建立一个结构体，每个结构体中存储三个信息，分别是映射前的原始数，映射后的数，和在nums出现的位置索引。

自定义排序规则cmp，先按照映射后的数排序，如果相同，再按照在nums出现的位置索引排序。

第三题-mid

2192. 有向无环图中一个节点的所有祖先

题目描述

给你一个正整数 n ，它表示一个 有向无环图 中节点的数目，节点编号为 0 到 n - 1 （包括两者）。

给你一个二维整数数组 edges ，其中 edges[i] = [fromi, toi] 表示图中一条从 fromi 到 toi 的单向边。

请你返回一个数组 answer，其中 answer[i]是第 i 个节点的所有祖先，这些祖先节点升序排序。

如果 u 通过一系列边，能够到达 v ，那么我们称节点 u 是节点 v 的祖先节点。

样例

  • 示例1
  • 示例2

思路

第一遍直接用弗洛伊德做的，复杂度是O(n3)O(n^3)O(n3)，最后超时了。

用n遍dijkstra复杂度降维O(n2log⁡(n))O(n^2\log(n))O(n2log(n)),就不超时了。
